## 三、飞书插件技能沉淀（本次深度学习）

### 飞书多维表格（Bitable）核心要点

**最容易踩坑的字段格式**：

| 字段 | 类型 | 正确格式 | 常见错误 |
|------|------|----------|----------|
| 日期 | type 5 | **毫秒**时间戳 | ❌ 传秒/字符串 |
| 人员 | type 11 | `[{"id": "ou_xxx"}]` | ❌ 传字符串/name字段 |
| 单选 | type 3 | `"选项名"` | ❌ 传数组 |
| 多选 | type 4 | `["选项1", "选项2"]` | ❌ 传字符串 |
| 超链接 | type 15 | `{"text": "显示", "link": "url"}` | ❌ 只传url字符串 |
| 附件 | type 17 | `[{"file_token": "xxx"}]` | ❌ 传外部url/本地路径 |

**操作流程最佳实践**：
1. 创建App后先删除默认表自带的空行
2. 写记录前**必须**先调用 `list_fields` 获取字段type
3. 批量操作单次 ≤ 500条，分批处理，串行写入
4. 不存在的单选/多选选项会自动创建，无需提前添加
