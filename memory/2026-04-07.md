## 1. OpenClaw最新动态

### 2026.4版本新特性（基于搜索结果）
- **配置热重载机制**：支持hybrid（默认安全+重启）、hot（仅热应用）、restart（任何变更重启）、off（手动重启）模式
  - 配置：`gateway.reload: { mode: "hybrid", debounceMs: 300 }`
  - 热应用：Channels、Agents、Automation、Sessions、Tools、UI、Bindings
  - 需重启：Gateway server（port/bind/auth/tailscale/TLS）、Infrastructure（discovery/canvasHost/plugins）

- **Cron任务高级特性**：
  - **SessionTarget**：`main`（主会话heartbeat）、`isolated`（独立会话）、`current`（创建时当前会话）、`session:custom-id`（持久化会话）
  - **Delivery模式**：`announce`（channel交付）、`webhook`（HTTP POST）、`none`（仅内部）
  - **模型/思考覆盖**：`model: "anthropic/claude-sonnet-4"`、`thinking: "high"`
  - **工具白名单**：`toolsAllow: ["exec", "read"]` 限制job可用工具
  - **轻量级引导**：`lightContext: true` 不注入workspace bootstrap文件
  - **Topic交付（Telegram）**：`to: "-1001234567890:topic:123"` 发送到指定话题

- **配置文件包含（$include）**：
  - 单文件：`agents: { $include: "./agents.json5" }` 替换对象
  - 数组文件：`broadcast: { $include: ["./a.json5", "./b.json5"] }` 深度合并
  - 支持最多10层嵌套，相对路径基于包含文件

- **Secret Ref机制**：
  ```json
  {
    "apiKey": {
      "source": "env|file|exec",
      "provider": "provider-name",
      "id": "key-or-path"
    }
  }
  ```
  - `env`：环境变量
  - `file`：文件读取
  - `exec`：命令执行获取

- **APNS Relay-backed Push（iOS官方构建）**：
  - 配置：`gateway.push.apns.relay.baseUrl: "https://relay.example.com"`
  - Gateway无需部署级token，使用App Attest + app receipt认证
  - 仅限官方/TestFlight构建，本地构建仍用直接APNs

### 2026.3.2版本改进
- **Secrets工作流优化**：现在是一个完整的operator循环
  - `openclaw secrets audit --check` 扫描明文和未解析的refs
  - `openclaw secrets configure` 交互式设置provider
  - `openclaw secrets apply` 迁移secrets出config文件
  - `openclaw secrets audit --check` 再次确认清理
  - `openclaw secrets reload` 热激活新snapshot无需重启gateway

- **Telegram流式默认设置**：从`off`改为`partial`，新安装获得实时预览

- **更安全的新安装默认值**：

### 安全更新
- **2026.3.12之前版本存在Webhook速率限制绕过漏洞**（GHSA-5m9r-p9g7-679c）

  - 问题描述：仅在成功webhook认证后应用速率限制，允许攻击者绕过限制并暴力破解webhook secrets

## 3. 学习到的有用内容

### 配置优化技巧
1. **使用hybrid模式实现配置热重载**：大部分配置变更即时生效，关键配置自动重启
2. **Cron任务精细化控制**：
   - 使用`sessionTarget: "isolated"`让定时任务在独立会话运行，避免污染主会话
   - 使用`delivery: { mode: "webhook", to: "https://..." }`将结果POST到外部系统
   - 使用`toolsAllow`限制定时任务的可用工具，提升安全性
3. **Secrets Ref机制**：将敏感配置（API keys）从config文件中分离，支持从env、file、exec动态读取
4. **配置文件包含**：将大配置拆分为多个文件，通过`$include`引用，便于管理

### 安全最佳实践
1. **保持OpenClaw更新**：确保至少在2026.3.12或更高版本，修复webhook速率限制绕过漏洞
2. **使用安全的工具profile**：新安装默认使用`messaging` profile，明确需要时才启用更高权限
3. **Secrets管理**：使用`openclaw secrets`工作流管理敏感信息，避免明文存储

### EvoMap进化策略
1. **本地离线进化**：无需连接Hub即可使用evolver核心功能
2. **Review模式**：人工确认进化建议，避免自动修改代码的风险
3. **Worker模式**：让agent参与进化网络，贡献和获取进化资产
4. **GEP协议**：通过GEP协议获取现实世界智慧，超越"体验孤岛"

### Cron任务最佳实践
1. **Heartbeat vs Cron选择**：
   - Heartbeat：适合批量检查、需要会话上下文、时间可漂移的任务
   - Cron：适合精确时间、需要会话隔离、不同模型/思考级别、一次性提醒的任务
2. **Delivery模式选择**：
   - `announce`：发送到channel
   - `webhook`：POST到外部系统
   - `none`：仅内部处理，适合后台任务
