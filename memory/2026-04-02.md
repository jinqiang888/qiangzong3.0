### 学习成果

#### 一、OpenClaw最新玩法和配置优化方案

**核心发现：**

**1. 部署优化**
- 使用 `openclaw@beta` 版本更稳定，Node.js 22.x 为必须版本
- 配置安全：`gateway.bind: "127.0.0.1"`（绝不使用 0.0.0.0）
- 端口优化：更改默认端口（18789）为随机高位端口
- 模型选择：设置 `agents.defaults.thinkingDefault: "medium"` 为全局默认

**2. Token 成本优化（直降 90%）**
- 使用 `qmd` 总结长线程/研究后再处理
- 简单任务路由到本地模型，重度推理使用云端模型
- 添加 Brave Search API Key 作为默认 web search

**3. ACP（Agent Client Protocol）高级玩法**
- 协议层本身消耗 **零 Token**
- 会话是非交互式的（无 TTY）
- 权限模式必须预先配置

**4. Session 自动重置（2026.3新功能）**
- 每日凌晨自动重置会话，保持上下文清爽
- 避免累积膨胀导致的性能下降

**5. 9大避坑技巧**
- 不要用 `npm install -g openclaw@latest`
- 不要使用 0.0.0.0 作为 bind 地址
- 定期备份 `~/.openclaw` 配置目录
- 环境变量存储在 `.env` 并设置 `chmod 600` 权限
- 远程访问通过 Tailscale/Cloudflare Tunnel，不直接端口转发
- 为非可信代理禁用高风险工具
- 定期运行 `openclaw security audit`
- 定期运行 `openclaw update` 获取最新技能
- PowerShell 中使用 `;` 分隔命令，不要使用 `&&`

#### 二、EvoMap 高质量进化资产和解决方案

**核心发现：**

**1. EvoMap 核心理念**
- Slogan："把最优解带给每个人"，"One agent learns, millions inherit"
- 解决 AI 经验无法共享的痛点
- 让一个 Agent 的经验可被百万 Agent 继承

**2. GEP 协议（Genome Evolution Protocol）**
三大核心要素：
- **Gene（基因）**：AI 在自然选择中沉淀的可复用方法论
- **Capsule（胶囊）**：直接可执行的能力包，灵感源自《黑客帝国》"脑后插管学功夫"
- **Event（事件）**：用于异步协作和进化触发

**3. 生态规模**
- 5.7万 Agent
- 47万胶囊资产
- 远超同类 Skill 商店

**4. 三层安全防护**
- 静态扫描 + 动态扫描 + 沙箱验证

**5. 自然选择机制**
- GDI 评分系统，从内在质量、实际使用数据、社交信号、新鲜度四个维度排名

**6. "线粒体战略"**
- 不追求垄断，做所有 Agent 的好朋友
- 坚持去中心化与开源精神

**7. AI 自进进化的三个阶段**
- 第一阶段：AI 自己写给自己用（14小时修复各种问题）
- 第二阶段：AI 写出来给其他 AI 用（Evolver 就是这个阶段）
- 第三阶段：AI 自传播（3.6万下载量，绝大部分是 AI 自动安装的）
