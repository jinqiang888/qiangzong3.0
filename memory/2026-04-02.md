### OpenClaw最新玩法与配置优化

#### 1. 配置热重载机制（重要新发现）
- **hybrid模式（默认）**：安全配置即时生效，关键配置自动重启
- **hot模式**：仅热应用安全配置，需要重启时输出警告
- **restart模式**：任何配置变更都重启
- **off模式**：禁用文件监听，需手动重启

**热应用无需重启的配置：**
- Channels所有配置
- Agents & models
- Automation（hooks, cron, heartbeat）
- Sessions & messages
- Tools & media
- UI & misc
- Bindings

**需要重启的配置：**
- Gateway server配置（port, bind, auth, tailscale, TLS, HTTP）
- Infrastructure（discovery, canvasHost, plugins）

**实用技巧：** `gateway.reload.debounceMs: 300` 防抖300ms避免频繁重启

#### 2. Cron任务高级特性

**SessionTarget新机制：**
- `main`：在主会话的下次heartbeat运行
- `isolated`：独立会话 `cron:<jobId>`，默认announce输出
- `current`：绑定到创建时的当前会话
- `session:custom-id`：持久化命名会话，保留上下文

**交付模式（Delivery）：**
- `announce`：直接通过channel adapter交付
- `webhook`：POST到指定URL
- `none`：仅内部运行，不交付

**模型和思考级别覆盖：**
```json
{
  "model": "anthropic/claude-sonnet-4-20250514",
  "thinking": "high"
}
```

**工具白名单（toolsAllow）：**
- 限制job只能使用特定工具：`["exec", "read", "write"]`
- 提升安全性，防止敏感任务越权

**轻量级引导上下文（lightContext）：**
- 不注入workspace bootstrap文件
- 适合高频低复杂度的定时任务

#### 3. 配置文件包含（$include）新特性
- 支持单个文件替换包含对象
- 支持数组文件深度合并（后覆盖前）
- 支持最多10层嵌套
- 相对路径基于包含文件解析

#### 4. 环境变量配置增强
- 支持 `.env` 和 `~/.openclaw/.env` 两级加载
- 支持内联env vars：`env.vars.GROQ_API_KEY`
- 支持shell env导入：`env.shellEnv.enabled: true`
- 支持配置值中env变量替换：`${VAR_NAME}`

#### 5. Secret Ref机制
支持三种secret来源：
- `env`：从环境变量
- `file`：从文件读取
- `exec`：通过命令执行获取

#### 6. APNS Relay-backed Push（iOS官方构建）
- 支持通过外部relay发送push通知
- Gateway不需要部署级relay token
- 使用App Attest + app receipt进行认证
