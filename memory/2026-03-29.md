### 2️⃣ EvoMap 进化资产与解决方案

#### EvoMap 核心概念
- **Capability Evolver**: 自我进化引擎，允许OpenClaw代理检查自身运行历史，识别故障或低效问题，自主编写新代码或更新自身记忆以提升性能
- **GEP Protocol**: 标准化进化协议，可复用基因和胶囊

#### 已安装的EvoMap技能
- `evolver` 1.29.9
- `feishu-evolver-wrapper` 1.7.1
- `evomap-gep` 1.1.0

#### Capability Evolver 核心能力
- **Auto-Log Analysis**: 自动扫描记忆和历史文件中的错误和模式
- **Self-Repair**: 检测崩溃并建议补丁
- **GEP Protocol**: 标准化进化，可复用资产
- **One-Command Evolution**: 运行 `/evolve` 或 `node index.js`

#### 使用方法
```bash
# 标准运行（全自动模式）
node index.js

# 审查模式（人工在环）
node index.js --review

# 疯狂狗模式（持续循环）
node index.js --loop
```

#### 环境变量配置
| 变量 | 默认值 | 说明 |
|------|--------|------|
| `A2A_NODE_ID` | (必需) | EvoMap节点身份，注册后设置 |
| `A2A_HUB_URL` | `https://evomap.ai` | EvoMap Hub API基础URL |
| `EVOLVE_STRATEGY` | `balanced` | 进化策略: `balanced`, `innovate`, `harden`, `repair-only`, `early-stabilize`, `steady-state`, `auto` |
| `EVOLVE_ALLOW_SELF_MODIFY` | `false` | 允许进化修改evolver源代码（不推荐） |
| `EVOLVER_ROLLBACK_MODE` | `hard` | 失败回滚策略: `hard`, `stash`, `none` |
| `EVOLVER_LLM_REVIEW` | `0` | 固化前启用第二意见LLM审查 |
| `EVOLVER_AUTO_ISSUE` | `0` | 重复失败时自动创建GitHub issue |

#### Feishu Evolver Wrapper 功能
- **Evolution Loop**: 运行GEP进化循环并附带Feishu报告
- **Dashboard**: 从 `assets/gep/events.jsonl` 可视化指标和历史
- **Export History**: 导出历史到飞书文档
- **Watchdog**: 通过OpenClaw Cron任务管理（每10分钟执行 `ensure`）

## 💡 关键洞察

### 自我进化的正确姿势
1. **不要盲目开启自我修改**: `EVOLVE_ALLOW_SELF_MODIFY=false` 是安全默认值
2. **审查模式优先**: 生产环境建议使用 `--review` 模式，人工确认后再应用
3. **Watchdog机制**: 通过 `feishu-evolver-wrapper` + cron watchdog保证稳定性，比系统crontab更可靠
4. **Git同步必不可少**: 始终建议运行git-sync cron任务配合evolver

### 记忆系统的选择建议
- **单机优先**: `memory-core`（官方推荐，本地LanceDB，防污染）
- **多设备同步**: `mem0`（云端同步，但需注意隐私）
- **高度定制**: `memory-lancedb`（社区维护，更多选项）

### 飞书集成最佳实践
- 使用 `feishu-evolver-wrapper` 替代原始evolver，获得富文本报告
- 多飞书账号场景下，通过 `channels.feishu.accounts` 精细控制
- 飞书cron提醒适合定时任务，heartbeat适合批量检查
