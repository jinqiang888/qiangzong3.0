## 备份策略建议

### 方案A：仅备份核心配置（推荐）

**备份内容：**
- `openclaw.json` - 主配置文件
- `workspace/SOUL.md` - 你的角色定义
- `workspace/IDENTITY.md` - 身份配置
- `workspace/USER.md` - 用户偏好
- `workspace/AGENTS.md` - Agent行为准则
- `workspace/TOOLS.md` - 工具配置
- `workspace/skills/` - 本地安装的技能
- `.ssh/` - SSH密钥（如需远程访问）

**排除内容：**
- `agents/main/sessions/` - 会话历史（运行时生成）
- `agents/main/agent/models.json` - 模型缓存
- `agents/main/agent/auth-profiles.json` - 认证缓存

### 方案B：全量备份（完整但冗余）

备份所有文件，包括运行时生成的会话数据。

## 使用步骤

### 1. 创建GitHub仓库

```bash
# 在GitHub上创建一个私有仓库（建议：openclaw-config-backup）
```

### 2. 配置SSH密钥（如使用SSH方式）

```bash
# 生成SSH密钥（如果还没有）
ssh-keygen -t ed25519 -C "openclaw-backup"

# 将公钥添加到GitHub账户
cat ~/.ssh/id_ed25519.pub
```

### 3. 修改备份脚本

编辑 `backup_github.py` 或 `backup_github_simple.bat`：

```python
# 修改这一行为你的GitHub仓库地址
REPO_URL = "git@github.com:YOUR_USERNAME/openclaw-config-backup.git"
```

### 4. 首次运行初始化

**Windows:**
```cmd
cd C:\Users\Administrator\.openclaw\workspace
backup_github_simple.bat
```

**Linux/Mac:**
```bash
cd ~/.openclaw/workspace
chmod +x backup_github_simple.sh
./backup_github_simple.sh
```

### 5. 设置定时备份

**Windows任务计划程序：**
1. 打开"任务计划程序"
2. 创建基本任务 → 名称："OpenClaw配置备份"
3. 触发器：每天/每周
4. 操作：启动程序
   - 程序：`C:\Users\Administrator\.openclaw\workspace\backup_github_simple.bat`
   - 起始于：`C:\Users\Administrator\.openclaw\workspace`

**Linux/Mac Cron:**
```bash
crontab -e

# 每天凌晨2点备份
0 2 * * * cd ~/.openclaw/workspace && ./backup_github_simple.sh >> backup.log 2>&1
```

## Cron Job配置示例

使用OpenClaw cron系统：

```json
{
  "name": "OpenClaw配置备份",
  "schedule": {
    "kind": "cron",
    "expr": "0 2 * * *"
  },
  "payload": {
    "kind": "systemEvent",
    "text": "请执行备份脚本: cd ~/.openclaw/workspace && python backup_github.py"
  },
  "sessionTarget": "main",
  "delivery": {
    "mode": "none"
  }
}
```

## 安全建议

1. **使用私有仓库** - 配置包含敏感信息（API密钥等）
2. **不要备份.env文件** - 避免泄露环境变量
3. **定期检查仓库** - 确保没有意外提交敏感数据
4. **使用.gitignore** - 排除临时文件和日志

### .gitignore示例
```gitignore
*.log
.env
*.tmp
.DS_Store
__pycache__/
*.pyc
```

## 故障排查

**问题：SSH权限拒绝**
```bash
# 测试SSH连接
ssh -T git@github.com
```

**问题：Git推送到远程失败**
```bash
# 检查远程配置
git remote -v

# 重新设置远程
git remote set-url origin git@github.com:YOUR_USERNAME/repo.git
```

**问题：文件路径包含中文**
- 建议使用英文路径或PowerShell的UTF-8模式

## 自动化选项

### 通过OpenClaw cron自动执行

每次有配置变更时自动触发备份（当前cron job的逻辑）。

优点：
- 精准检测变更
- 自动触发备份
- 无需手动操作

缺点：
- 频繁提交可能产生过多commit
- 需要GitHub访问权限
