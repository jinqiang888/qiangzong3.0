### 一、OpenClaw 最新玩法与技巧汇总

#### 1. Cron 任务高级玩法
- **4 种会话绑定模式**：
  - `main`：主会话，适合需要全局上下文的任务
  - `isolated`：隔离会话，独立执行不干扰主上下文，默认自动推送结果
  - `current`：绑定当前创建任务的会话
  - `session:custom-id`：自定义持久会话，保留上下文支持多轮任务
- **时间精度控制**：
  - 支持 `at`（单次）、`every`（间隔）、`cron`（表达式）三种调度方式
  - 新增 `staggerMs` 配置，避免整点任务集中触发负载过高
  - 自定义时区支持，解决跨时区调度问题
- **重试策略优化**：
  - 临时错误（限流、网络波动）自动指数退避重试（30s → 1m → 5m）
  - 永久错误（配置错误、认证失败）立即禁用
  - 支持自定义重试次数和间隔

#### 2. Skills 系统优化配置
- **技能优先级体系**：
  - 工作区技能 (`<workspace>/skills`) > 全局技能 (`~/.openclaw/skills`) > 内置技能
  - 插件可自带技能，启用插件时自动加载
- **按需加载机制**：
  - 通过 `metadata.openclaw.requires` 配置依赖，自动检测环境可用性
  - 支持 OS 过滤、二进制依赖检查、环境变量校验
  - 未满足依赖的技能自动隐藏，不占用系统提示词 Token
- **性能优化**：
  - 会话启动时快照技能列表，后续轮次复用
  - 技能变更自动热重载，无需重启服务
  - 每个技能仅占用 ~24 Token，62 个技能总开销 < 1500 Token

#### 3. 实用配置技巧
- **轻量化任务模式**：`lightContext: true` 跳过工作区文件注入，大幅提升定时任务启动速度
- **多渠道消息投递**：直接配置 `delivery.channel` 支持 Telegram/Discord/飞书/WhatsApp 等多渠道推送
- **自定义会话保留时间**：`cron.sessionRetention` 配置历史会话清理周期，节省存储空间
