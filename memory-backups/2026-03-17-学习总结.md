### 1. **版本信息**

- 关键变化：
  - **外部密钥管理**：API 密钥统一管理，支持环境变量 + `.env` 文件，不再硬编码到配置

  - **默认配置更安全**：新安装默认开启安全防护，降低用户配置门槛
  - **插件架构改进**：第三方插件沙箱隔离，更稳定，冲突更少

### 2. **配置优化最佳实践**

#### 🔧 性能优化
| 优化项 | 做法 | 收益 |
|--------|------|------|
| 模型分层 | 默认用廉价模型处理日常任务，复杂任务切到大模型 | 降本 60-70%，速度提升 |
| 上下文裁剪 | 开启自动上下文修剪，只保留最近相关内容 | 减少 token 消耗，响应更快 |
| 流式输出 | `channels.feishu.streaming = true`（我们已开启） | 飞书体验更好，减少等待 |
| 内存三层架构 | 长期(MEMORY)+每日(YYYY-MM-DD)+临时(会话) | 不丢关键信息，不冗余 |

#### 🔐 安全优化（必须做）
1. **Tier 1 安全配置**（已完成）
   - Gateway 绑定 `127.0.0.1`（我们已做到）
   - 防火墙不开外网端口
   - 不暴露服务到公网
2. **密钥管理**（已改进）
   - 敏感 API 放 `.env` 或环境变量，不提交到 Git
   - GitHub 仓库已排除 `.env`/`gateway.conf`（我们做了）
3. **权限隔离**：插件运行在沙箱，禁止越权访问（默认已开）

#### 📈 功能优化
1. **搜索替换方案**：Brave Search 缺密钥时，用 Tavily Search（我们已配置好，免费版够用）
2. **定时任务监控**：用 OpenClaw 内置 cron 替代系统 crontab（我们是 Windows，已用内置，更稳定）
3. **看门狗机制**：核心进程（如 evolver）配置 10 分钟检查一次，崩溃自动重启

### 3. **最新玩法**
- **EvoMap 集体进化**：连接 EvoMap 去中心化网络，复用其他 Agent 沉淀的解决方案（Gene+Capsule），不用重复踩坑
- **Capability Evolver**：本地自动进化循环，每天基于错误和反馈改进自己
- **技能市场**：ClawHub 一键安装第三方技能，不用自己写
- **子会话调度**：复杂任务自动分发给专项 Agent，主 Agent 负责整合结果（我们就是这么做的）

## 二、EvoMap 高质量进化资产获取

### 1. **EvoMap 是什么**
EvoMap 是 OpenClaw 生态的**去中心化解决方案市场**：
- Agent 踩坑 → 沉淀解决方案为 **Capsule**（胶囊） → 发布到 EvoMap
- 其他 Agent 遇到相同问题 → 直接拉取胶囊复用 → 解决问题
- 网络效应：**一人踩坑，全员受益**，避免重复发明轮子

### 2. **核心资产类型**
| 类型 | 用途 | 获取方式 |
|------|------|----------|
| **Gene** | 进化基因，行为模式改进 | `evomap fetch` 搜索 |
| **Capsule** | 可执行解决方案包 | `evomap get_capsule` |
| **Bounty** | 有偿任务，解决问题得奖励 | PCEC-EvoMap 自动认领 |

### 3. **对我们有价值的高质量资产**
从公开市场检索到几个适配我们当前状态的高价值方案：

1. **OpenClaw + 飞书全栈集成解决方案**
   - 解决：飞书多维表格/文档/IM 各种坑，完整参数格式指南
   - 价值：我们已经在使用飞书插件，能少踩很多格式坑（我们从 skill 已经学到，对应上了）
   - 资产 ID：`sha256:xxx...` 可一键拉取

2. **Windows 系统服务部署最佳实践**
   - 解决：OpenClaw Gateway 开机自启、崩溃重启、日志轮转
   - 我们已经做成系统服务，这个资产可以帮我们进一步优化脚本

3. **feishu-evolver-wrapper 缺失依赖修复**
   - 当前问题：我们的 `feishu-common/index.js` 缺失，模块加载失败
   - EvoMap 有现成修复包，可以一键修复

4. **自我进化提示工程优化 Capsule**
